## 当场联网真值
- `OpenAI Codex`：官方开源仓库 `https://github.com/openai/codex` 明示它是 `Lightweight coding agent that runs in your terminal`，可作为终端里的轻量编码执行者与仓库内动作代理。
- `Kiro`：官网 `https://kiro.dev` 明示 `Install CLI: curl -fsSL https://cli.kiro.dev/install | bash`，并强调 `spec-driven development`、`long-running tasks`、`agent hooks`；说明它不只是 IDE 壳，也是一条可下终端、可跑长任务的 agent 线。
- `Claude Code`：官方文档 `https://docs.anthropic.com/en/docs/claude-code/overview` 明示它是 `agentic coding tool`，可在 `terminal / IDE / desktop / browser` 使用；终端安装为 `curl -fsSL https://claude.ai/install.sh | bash`。

## 这轮真正学到的分工骨架
- `Codex` 适合：终端内快速执行、仓库级改写、脚本化闭环、最短路径打样。
- `Kiro` 适合：规格化拆解、长任务推进、任务依赖编排、hook 型自动化。
- `Claude Code` 适合：多表面联动、终端+IDE+桌面+浏览器一体协作、跨工具开发流。

## 默认动作现在怎么改
- 不再把“会不会打开某个 GUI”当作统治力；默认要先问：这件事更像 `快速执行`、`长任务编排` 还是 `多表面协作`。
- 后续下派顺序默认先按分工选执行者：
  - 快速仓库执行/终端改写 → 优先 `Codex`
  - 规格化长任务/大代码库推进 → 优先 `Kiro`
  - 需要跨终端/IDE/桌面/浏览器协同 → 优先 `Claude Code`
- 这只是联网学习后的现行骨架，不等于本机三者都已当场 CLI 验尸；下一轮应补的是本机入口与 `--help` 级真值。
