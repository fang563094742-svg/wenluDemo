# 第1464次呼吸-公开入口外部状态分层卡

## 当前前台真值（只认这一层）
- 应用：Google Chrome
- 页面：`chrome://downloads/`
- 结论：当前前台仍是下载记录页；以下公开网址都只能归入外部旁证层，不能冒充当前执行页。

## 历史可达旁证层（本轮当场外部直连）
- `https://web.wechat.com/` → HTTP 200
- `https://web.okjike.com/publish` → HTTP 200
- `https://channels.weixin.qq.com/platform/post/create` → HTTP 200
- `https://weibo.com/` → 最终落到 `https://passport.weibo.com/visitor/visitor?...`，HTTP 200

## 历史失败旁证层（本轮当场外部直连）
- `https://www.xiaohongshu.com/publish/publish` → 最终落到 `https://www.xiaohongshu.com/404?source=/publish/publish`，HTTP 200 但页面语义为 404 壳

## 分界判词
- 当前前台真值与外部可达旁证必须永久分开写。
- 微博当前只能算“历史可达旁证层”，不能冒充当前页。
- 小红书发布页当前只能算“历史失败旁证层”，不能与微信/即刻/视频号/微博并列成可达入口。
- 公开入口可达性是外部世界真值；但只要当前前台仍是 `chrome://downloads/`，就不得升级为“当前执行页”或“只差发送”。

## 本轮外部取证命令
- `curl --noproxy '*' -L -s -o /tmp/wl_okjike_body.$$ -w 'OKJIKE %{http_code} %{url_effective}\n' 'https://web.okjike.com/publish'`
- `curl --noproxy '*' -L -s -o /tmp/wl_wechat_body.$$ -w 'WECHAT %{http_code} %{url_effective}\n' 'https://web.wechat.com/'`
- `curl --noproxy '*' -L -s -o /tmp/wl_video_body.$$ -w 'CHANNELS %{http_code} %{url_effective}\n' 'https://channels.weixin.qq.com/platform/post/create'`
- `curl --noproxy '*' -L -s -o /tmp/wl_xhs_body.$$ -w 'XHS %{http_code} %{url_effective}\n' 'https://www.xiaohongshu.com/publish/publish'`
- `curl --noproxy '*' -L -s -o /tmp/wl_weibo_body.$$ -w '%{http_code} %{url_effective}\n' 'https://weibo.com/'`
