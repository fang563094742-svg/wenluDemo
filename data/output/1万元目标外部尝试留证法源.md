# 1万元目标外部尝试留证法源

更新时间：2025-06-09

## 目标
把每一次外部尝试、报价、承接、失败都压成可回看、可核对、可追责的证据；以后不再让“聊过了、快成了、在推进”这种口头话术冒充结果。

## 唯一账本
- 主账本：`data/attempt-ledger/attempt-ledger.json`
- 可读版：`data/attempt-ledger/attempt-ledger.md`
- 填写模板：`data/attempt-ledger/attempt-template.md`
- 生成命令：`npm run attempt:ledger`

## 什么叫一条有效记录
至少同时满足：
- 有外部对象：谁、在哪个渠道。
- 有需求摘要：对方要什么，不超过两句话。
- 有阶段：`attempted / quoted / accepted / failed / delivered`。
- 有结果：`pending / won / lost` 三选一。
- 有证据：聊天截图、报价文案、付款截图、交付文件、备注之一，必须给路径。

缺任何一项，都不算结果，只算待补证。

## 四个强制时点
- 一触达就记：新开口、新私聊、新投稿、新投递，立刻记一条 `attempted`。
- 一报价就补：金额、范围、周期、报价证据路径必须补齐。
- 一承接就改：对方确认或付定金，改成 `accepted`，并附到账证据。
- 一失败就写死因：必须写清失败原因，例如没预算、没回复、需求不明确、信任不足、价格过高、时效不匹配。

## 禁止口头进展
以下表述一律无效：
- “在推进”
- “快成了”
- “对方有兴趣”
- “聊得还可以”
- “后面再跟”

除非能在账本里对应到具体记录、具体报价、具体证据路径。

## 失败原因最小分类
- `无回复`：发出后无任何有效回应。
- `预算不匹配`：对方明确嫌贵或预算不足。
- `需求模糊`：三问补不齐，无法形成最小报价。
- `信任不足`：对方担心能力、案例、付款安全。
- `时效不匹配`：交付时间对不上。
- `范围过大`：需求超出当前最小承接边界。
- `已被他人承接`：对方已找别人。
- `主动放弃`：判断投入产出不值，主动止损。

## 执行纪律
- 每次对外动作结束后 3 分钟内补账本。
- 每天回看 `data/attempt-ledger/attempt-ledger.md`，只认账本里的推进。
- 周复盘只看三件事：尝试数、报价数、失败原因分布。
- 如果一周内仍出现口头进展但无账本记录，视为执行失守。
