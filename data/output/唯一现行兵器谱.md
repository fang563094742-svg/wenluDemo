## 3. 三大入口现行结论

### 3.1 Codex：当前主执行器，入口最完整
**当前可调度入口**
- CLI：`/Applications/Codex.app/Contents/Resources/codex`
- Desktop：`Codex.app`

**本轮已实测成功**
- `codex --help`：可列出完整子命令面。
- `codex doctor`：可返回安装、认证、配置、MCP、sandbox 健康信息。
- `codex exec --skip-git-repo-check --sandbox workspace-write 'Reply with exactly: CODEX_EXEC_OK' </dev/null`：**非交互执行成功**。
- `osascript -e 'tell application "Codex" to activate'`：桌面应用可被前台化。
- `codex mcp list`：可列出现行 MCP 服务器。

**当前边界**
- 认证已通，但当前为 **API key auth**；远程插件同步会报 `chatgpt authentication required`，说明**本机执行可用，不等于远程插件生态全通**。
- `codex exec` 若不给 `</dev/null`，会提示 `Reading additional input from stdin...`，说明**非交互调度时需显式处理 stdin**。
- 已见现行 MCP：`computer-use`、`node_repl`、`playwright`；但其中启动/握手存在超时或 `Unsupported` 标识，说明**列得出不等于本轮已打通全部 MCP 闭环**。

**判级**
- **A级主力**：默认总调度器 + 主执行器。

### 3.2 Kiro：当前主要是编辑器/文件跳转入口
**当前可调度入口**
- CLI：`/usr/local/bin/kiro`
- Desktop：`Kiro.app`

**本轮已实测成功**
- `kiro --help`：可列出 `goto/diff/merge/add` 等编辑器控制命令。
- `kiro --goto package.json:1:1 --reuse-window`：命令退出码 `0`，文件跳转入口可用。
- `kiro --list-extensions --show-versions`：可列出扩展（本轮观察到中文语言包）。
- `osascript -e 'tell application "Kiro" to activate'`：桌面应用可被前台化。

**当前边界**
- 现行 CLI 能力集中在**打开、定位、比对、合并、扩展管理**。
- 本轮**未发现**像 Codex 那样的 agent 非交互执行面；因此它现在更像**可被调起的编辑器前台**，不是独立主执行器。
- 运行环境里 `TERM_PROGRAM=kiro`，说明当前这条任务线本身就运行在 Kiro 终端壳里，但这不等于 Kiro 具备额外自动代理能力。

**判级**
- **A-/B+ 辅助主力**：适合作为可视编辑器、定位器、冲突处理位。

### 3.3 Claude：当前只确认桌面拉起与应用级控制
**当前可调度入口**
- Desktop：`Claude.app`
- Bundle ID：`com.anthropic.claudefordesktop`
- CLI：**当前 PATH 未见 `claude`**

**本轮已实测成功**
- `open -a Claude`：应用可拉起。
- `osascript -e 'tell application "Claude" to activate'`：应用可前台化。
- `System Events` 可观测到 `Claude` 进程存在。

**当前边界**
- 当前**没有确认到 Claude CLI**，所以不存在可直接 shell 调度的稳定命令面。
- 本轮也**没有拿到可脚本化对话接口**；仅能确认桌面应用层面的拉起、激活、进程观测。
- 因此现阶段 Claude 适合放在**人工接管/人工评审/备用脑位**，不应写成强依赖自动执行链。

**判级**
- **B级备用位**：能打开、能切前台，但还不是本轮可自动化主力。

## 4. 统一边界判词
- **最能直接被 shell 调度的是 Codex**：既有 CLI、诊断、非交互执行，也有桌面应用入口。
- **Kiro 当前是编辑器控制面，不是代理执行面**：文件跳转、diff、merge 已通，但没有验证到独立 agent 闭环。
- **Claude 当前只有桌面入口，没有现成 CLI 调度面**：应降级为人工评审位，不写进自动执行关键路径。

## 5. 默认现行分工
- `Codex`：接总目标、拆分、执行、产物落盘。
- `Kiro`：接明确文件定位、差异查看、人工编辑接管。
- `Claude`：只在高不确定性判断或文字润色时作为备用脑。
- `shell + osascript + open`：任何桌面入口失灵时的最终兜底。

## 6. 最小复验命令
```sh

kiro --help | sed -n '1,40p'
kiro --goto package.json:1:1 --reuse-window; echo $?
open -a Claude && osascript -e 'tell application "Claude" to activate'
```
