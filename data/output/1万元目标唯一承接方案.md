## 目标
把收款承接链压到一个唯一方案：最小暴露、最少动作、可直接收大额或分段定金，目标口径按单笔/分段累计打到 10000 元。

## 唯一默认方案
**默认只走：微信/支付宝收款码直收 + 50% 定金锁排期 + 大额拆两段。**

不用公开页，不用 localhost，不用币安做默认入口，不先解释主体身份。

## 为什么这是唯一默认方案
1. **动作最少**：对方确认后，只需发一句收款话术 + 一张收款码。
2. **暴露最小**：默认不报姓名，只有对方明确核验时才回复“方海弟”。
3. **可收大额**：
   - 300~3000：直接全款或 50% 定金。
   - 3000~10000：默认 50% 定金 + 50% 首版前/验收前。
   - 对方单次支付受限：拆成多笔定金，口径仍按同一项目累计。
4. **现成可用**：本地已有 `data/payment-assets/wechat-pay.jpg` 与 `data/payment-assets/alipay-pay.jpg`，无需新开户。
5. **风险更低**：币安仅作客户主动提出时的备选，不作为默认收口。

## 唯一动作链
1. 对方需求一来，只收三件事：`输入是什么 / 想输出什么 / 最晚什么时候要`。
2. 信息够，直接回：`能做 + 周期 + 报价 + 先付50%定金锁排期`。
3. 对方点头，直接问：`微信还是支付宝？`
4. 发对应收款码。
5. 对方付款后回截图。
6. 确认到账，回复：`已确认到账，锁排期，24~48小时给首版。`
7. 大额单默认在话术里写明：`这单按两段收，先50%定金，首版确认后补尾款。`
