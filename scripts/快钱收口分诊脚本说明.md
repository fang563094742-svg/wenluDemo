# 快钱收口分诊脚本说明

目标：把“收到线索 → 给出首条收口回复 → 直接推进报价/追问/定金”压成一条命令，优先服务当天小单收口。

## 入口

```bash
npx tsx scripts/dealAccelerator.ts "客户原话/需求描述"
```

## 会自动产出

- `artifacts/deal-accelerator-时间戳/analysis.json`
- `artifacts/deal-accelerator-时间戳/reply.txt`
- `artifacts/deal-accelerator-时间戳/clipboard.txt`
- `artifacts/deal-accelerator-时间戳/summary.txt`
- `artifacts/deal-accelerator-时间戳/payment_reply.txt`
- `artifacts/deal-accelerator-时间戳/payment_request.txt`
- `artifacts/deal-accelerator-时间戳/receipt_confirmation.txt`
- `artifacts/deal-accelerator-时间戳/kickoff_confirmation.txt`
- `artifacts/deal-accelerator-时间戳/成交加速包.md`

## 这版新增能力

不再只是“识别类别 + 报价”，而是补成一步式成交加速链：

- 自动识别平台语境：微信 / 小红书 / V2EX / 抖音 / 视频号 / 通用
- 生成平台适配首回消息
- 生成一步式报价推进文案
- 生成二次追问推进文案
- 继续保留收款回复 / 催付口径 / 到账确认 / 开工确认
- 小单默认优先直收定金锁排期；陌生来源再补平台留痕

## 当前最短动作链

1. 复制客户原话
2. 跑命令
3. 直接粘贴 `clipboard.txt`（已自动写入剪贴板）
4. 如果客户继续问，就从成交包里复制对应“报价推进 / 追问 / 收款”口径

## 三种典型输出

### 1. 信息齐全的小快单
脚本会直接给：
- 可接判断
- 建议交付
- 周期
- 报价
- 平台适配首回
- 一步式报价推进文案

### 2. 信息不齐但仍可推进
脚本会自动压成最多 3 个关键追问，优先补：
- 输入
- 输出
- 截止

同时补出平台适配追问首回，避免空聊。

### 3. 伪快单/系统单
脚本会直接给“明确不做”或“先压 MVP”的回复，不让动作链被拖长。

## 示例

```bash
npx tsx scripts/dealAccelerator.ts "小红书来的，想找人做个本地批量改标题工具。我这边已有商品图、原始标题和文案表，想导出新标题csv，最好明晚前能用。"
```

## 推荐使用法

- 小红书 / 抖音 / 视频号：优先用平台适配首回，语气更短更利于转私聊
- 微信 / 熟人转介绍：若信息已全，直接用报价推进文案
- 陌生公开流量：若你想先留痕，再用收款回复里的平台入口页口径
