# 成交压缩能力说明

更新时间：2026-06-07
目标：把现有成交/服务单页，自动压成更短的即发版平台帖与私聊首回，并沉淀为可复用动作。

## 新能力入口

```bash
npm run copy:compress -- "今晚直接外发收定金的一页成交包.md"
```

也可替换成任意现有成交单页：

```bash
npm run copy:compress -- "今晚最短可直接复制发送版.md"
npm run copy:compress -- "cash-path-总成交包.md"
```

## 会自动产出

每次运行都会在 `artifacts/copy-compressor-时间戳/` 下生成：

- `platform-post.txt`：平台即发短帖
- `private-opening.txt`：私聊首回
- `rules.txt`：本次抽出的压缩规则
- `analysis.json`：结构化抽取结果
- `成交压缩包.md`：可直接阅读的单文件结果

## 当前压缩规则

1. 先抽一句话卖点：只留“帮谁把什么重复工作压成什么结果”。
2. 平台短帖只保留四块：对象/痛点、可做范围、三问动作、结果承诺。
3. 私聊首回只做成交分诊，不讲技术细节，优先收“输入 / 输出 / 截止”。
4. 价格、周期、定金、修改次数优先复用已有锚点，不重新发明。
5. 超过字数时，先删举例，再删解释，最后保留三问与承诺。
